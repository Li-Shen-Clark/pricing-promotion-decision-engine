# `data/processed/` — schema & grain reference

Generated by the notebook pipeline in the full development workspace. This GitHub demo repo includes only the lightweight runtime artifacts required by the Streamlit app and tests. Large reconstruction/modeling panels are documented below for reproducibility but intentionally **not committed**; regenerate them locally by downloading the raw DFF files and rerunning the notebooks.

---

## `sku_store_week_panel.parquet` — robustness panel

- **GitHub demo repo**: documented only; not committed because the app does not read it at runtime.
- **Grain**: `UPC × STORE × WEEK` (one row per UPC-store-week after quality filters)
- **Rows**: 4,654,156  ·  **Size on disk**: 31.6 MB
- **Filters applied**: `OK = 1` ∧ `PROFIT ∈ [0, 99)` ∧ `PRICE > 0` (retained 70.5% of raw rows)
- **Produced by**: `01_data_cleaning.ipynb`

| Column | Type | Definition |
|---|---|---|
| `STORE` | int16 | DFF store id |
| `UPC` | int64 | UPC (product code = last 5 digits) |
| `WEEK` | int16 | DFF internal week index (1–400) |
| `week_start_date` | datetime64[ns] | Week 1 epoch = 1989-09-14 + (WEEK-1)×7 days |
| `year` / `month` | int / int8 | Calendar year/month of week start |
| `PRICE` | float32 | Bundle retail price (USD) |
| `QTY` | int8 | Bundle size (usually 1) |
| `MOVE` | int32 | Units sold that week |
| `SALE` / `sale_code_raw` | category / string | Original SALE code including G, L |
| `sale_type_clean` | category | `bonus_buy` / `coupon` / `price_reduction` / `unknown_promo` / `missing` (see rawData/README §3.3) |
| `promo` | bool | `sale_type_clean != 'missing'` |
| `PROFIT` | float32 | Gross margin (%) from DFF AAC |
| `unit_price` | float32 | `PRICE / QTY` |
| `unit_cost` | float32 | `unit_price × (1 - PROFIT/100)` |
| `revenue` | float32 | `unit_price × MOVE` |
| `COM_CODE` | int32 | DFF commodity code (Cereals = 311, single-valued) |
| `DESCRIP` | string | Raw product description (≤20 chars) |
| `SIZE` | string | Raw size text |
| `NITEM` | int64 | DFF item tracking code |

## `brand_size_store_week_panel.parquet` — baseline panel

- **GitHub demo repo**: documented only; not committed because the app uses frozen runtime summaries.
- **Grain**: `brand_final × size_oz_rounded × STORE × WEEK`
- **Rows**: 2,690,548  ·  **Size on disk**: 71.9 MB
- **Aggregation source**: the UPC panel filtered to `size_kind ∈ {oz, oz_bundle}` ∧ `brand_confidence ∈ {high, medium}` (96.6% of UPC rows)
- **Produced by**: `02_eda.ipynb`

**All MOVE-weighted aggregations** (see 02_eda §8 for the rules):

| Column | Definition |
|---|---|
| `brand_final` | Output of DESCRIP rule + manufacturer_code block cross-validation (02_eda §4) |
| `size_oz_rounded` | `round(size_oz, 2)` — avoids float-key splits at groupby |
| `STORE`, `WEEK`, `week_start_date` | Time / store keys |
| `quantity` | `Σ MOVE` |
| `revenue` | `Σ (unit_price × MOVE)` |
| `unit_price` | `revenue / quantity` (MOVE-weighted) |
| `unit_cost` | `cost_total / quantity` (MOVE-weighted) |
| `weighted_profit_margin` | `Σ (PROFIT × MOVE) / quantity` |
| `price_per_oz` | `unit_price / size_oz_rounded` |
| `promo_any` | `any(promo)` — **primary promo regressor** |
| `promo_share` | `Σ MOVE[promo] / quantity` — robustness only |
| `dominant_sale_type_all` | MOVE-weighted modal sale_type across all rows in cell |
| `dominant_promo_type` | MOVE-weighted modal sale_type within promo rows only (missing if no promo) |
| `n_upcs_in_cell` | Distinct UPCs aggregated into this cell |
| `n_promo_upcs` | Distinct UPCs in promo rows within this cell |
| `bundle_units` | Distinct QTY values within cell (>1 triggers `mixed_bundle` flag) |
| `unit_price_min` / `_max` / `_std` / `_cv` | Within-cell dispersion |
| `cell_quality_flag` | Comma-joined reasons (`low_quantity` / `mixed_bundle` / `high_price_dispersion` / empty = clean) |

## `brand_mapping.csv` — UPC → brand lookup

Produced by `02_eda.ipynb §4`. 486 rows, one per UPC, with:
`UPC, manufacturer_code, DESCRIP, descrip_clean, SIZE, size_oz, size_kind, brand_rule, block_majority_brand, brand_final, brand_source, brand_confidence`.

**Source taxonomy**: `descrip_rule` (matched DESCRIP keyword) · `manufacturer_block` (DESCRIP unknown, inferred from manufacturer_code majority) · `unknown` (neither).

**Confidence taxonomy**: `high` (rule ∧ block agree) · `medium` (one of rule/block available, the other missing) · `low` (rule/block conflict OR both unknown). Baseline modeling uses only `high ∪ medium`.
